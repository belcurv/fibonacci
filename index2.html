Each rect's width and height are always the same and must be positive numbers.

The x and y coordinates vary and are funky. What's the rhyme/reason? Depends on direction. Assuming counter-clockwise assembly:

  • UP   : x: -(previous LEFT box's 'x' value), y: -(width + previous LEFT box's 'y' value)
  • LEFT : x: -(width + previous LEFT box's 'x' value) , y: previous UP box's 'y' value
  • DOWN : x: previous LEFT box's 'x' value , y: previous LEFT box's 'y' value + previous LEFT box's 'height'. NOTE: the 1st element is a DOWN square that breaks the rule. It's coords are 0,0, dimensions: 1x1.
  • RIGHT: x: previous DOWN box's 'x' value + previous DOWN box's width , y: previous LEFT box's 'y' value
